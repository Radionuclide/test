Installation

Have ibaAnalyzer installed and registered as a COM-component, the version of ibaAnalyzer should be at least 5.0 To enable conditional tasks, the ibaAnalyzer version should be at least 5.4. To enable monitoring ibaAnalyzer for memory use and duration, the ibaAnalyzer version should be at least 5.8.2
Also install the .NET 2.0 framework, available from www.microsoft.com and
http://www.iba-ag.com/pub/ftp/ibaGent/2.0framework/dotnetfx.exe
Execute ibaDatCoordinatorInstall_v1.9.7.exe and follow instructions.

Version history

Version 1.9.7 - 12/03/2009

Enhancements

  • Fixed installer and build options so ibaDatCoordinator can be installed and runs on Windows 2008 64 bit server .
  • Uses the new ibaFiles with the new ibaFiles installer.

Bugfixes

  • When the service was stopped, a watchdog could be started, resulting in two watchdogs when the service restarted.

Version 1.9.6 - 22/10/2008

Bugfixes

  • On occasion the client crashed when applying the changes to a job configuration.
  • If less than 10 old .dat files were available for processing, processing didn't start immediately after the job was started.

Version 1.9.5 - 25/09/2008

Test version with more logging to test the cleanup strategy

Version 1.9.4 - 19/06/2008

Enhancements

  • The client program is made more economic on memory
  • Option to have the report, extract and copy/move tasks to simply overwrite target files if allready existant instead of adding an index to the filename.

Version 1.9.3 - 16/06/2008

Enhancements

  • When cyclic rescanning for unprocessed .dat files is enabled, the number of files that can be added to the list of .dat files to process is now limited to 10000.
  • The algorithm to cleanup files when a filesize quota is set (for tasks generating files as output) has been improved in performance.

Bugfixes

  • The algorithm to cleanup files failed when the output directory was the disks root directory.
  • A crash occasionally happened when updating a running job.

Version 1.9.2 - 16/04/2008

Enhancements

  • Extended the logging for the Copy/Move task.

Version 1.9.1 - 9/04/2008

New features

  • Copy/Move task extended to a Copy/Move/Delete task.
  • Watchdog extended so that you can select to have a binary message (with data at fixed offsets) instead of a text message. The text message itself is extended to provide information whether or not the job is started and the number of files which ibaDatCoordinator has tried to process the set maximum number of times without success.

Bugfixes

  • The current file the DatCoordinator was processing was sometimes not visible in status view.

Version 1.9.0 - 7/04/2008

Enhancements

  • Initial scans and cyclic scans for unprocessed .dat files are now run on a separate thread so that the execution of tasks is not halted.
  • No instances of ibaAnalyzer are started anymore when there are no tasks defined requiring ibaAnalyzer.

Bugfixes

  • You could delete tasks when the job was running. This should have been disabled.
  • Cleanup strategy of file output producing tasks was faulty when limiting the used diskspace to an amount over 2 GByte.

Version 1.8.10 - 12/02/2008

Enhancements

  • You can select to have IbaDatCoordinator restarting ibaAnalyzer after a certain number of times ibaAnalyzer has done a report or extraction or evaluated an expression. This is to free any memory leaked by ibaAnalyzer or one of its components.
  • You can select to have IbaDatCoordinator closing ibaAnalyzer if there are currently no files left to process. This way there is only an ibaAnalyzer process in memory when there are files to process.

Bugfixes

  • There was a problem with testing a script in the script configuration control.

Version 1.8.7 - 30/11/2007

Bugfixes

  • New version of ibaFiles included, which allows longer technostrings/infofields (up to 10000 charachters) not only to be read, but also to be written.

Version 1.8.6 - 30/11/2007

Bugfixes

  • Selecting the .dat directory, the folder dialog wasn't initialised to the previous path.

Version 1.8.5 - 29/11/2007

Bugfixes

  • In the ibaAnalyzer task control panes, there was a problem opening ibaAnalyzer if the analyzer path contained spaces.
  • New version of ibaFiles included, which allows longer technostrings/infofields (up to 10000 charachters).

Version 1.8.4 - 23/10/2007

New features

  • When generating files with the report task or extract task and with the copy task, you can select to clean up older generated output files according to a quota instead of limiting the number of generated directories.

Enhancements

  • The periodic notifications are altered so that the first notification is sent immediately if there hasn't been a notification before it in the selected time interval, in previous versions the notification was only sent after the selected time interval was passed.
  • The option to skip the first initial scan for unprocessed .dat files is moved from the service settings to the job level.

Bugfixes

  • After stopping the service, mimimizing the client and then restarting the service, the client would not reconnect to the service.
  • In the status pane, when viewing permanently failed .dat files, trying to go to the job configuration panel through the link caused an unhandled exception.

Version 1.8.3 - 11/09/2007

Bugfixes

  • Shortcut to logfiles fixed in the start menu.

Version 1.8.2 - 18/07/2007

New features

  • Scripts, the batchfile task is now a general script task. Syntax highlighting is available for Visual Basic Script and JavaScript.
  • Input parameters for the scripts are similar to postprocessing arguments in PDA (%f,%g,%h).
  • The copy task and script task can now work on the result of a previous task (e.g. generated report) instead of the original .dat file.
  • A test button is added to the notification options of a job.

Bugfixes

  • If datFiles had the exact same timestamp, one of them was not visible in the statusview.

Version 1.8.1 - 10/07/2007

New features

  • The number of times the DatCoordinator attempts to process a .dat file can be limited. After this limit is exceeded, such .dat files are put in a seperate list which the user can clean by selecting to either delete the .dat files or reset the info fields from the .dat files so the DatCoordinator treats them as newly generated files.
  • For a report, extract or conditional task it can be selected to have ibaAnalyzer monitored for memory consumption and duration of the task. When the time limit or memory limit is exceeded, the task will be terminated.
  • A new settings tab is added where the user can select a couple of options
    • You can select whether or not the DatCoordinator service is automatically started when windows starts.
    • A time interval can be specified to wait after windows startup before starting to process the .dat files.
    • The process priority of the service or application can be specified.
    • You can select to skip the initial scan for .dat files allready existing before a DatCoordinator job was running.
      In this case only .dat files created while the job is running will be considered for processing.

Enhancements

  • The DatCoordinator now tries to connect periodically to the service instead of going permanently in offline mode.
  • The location of the logfiles is now fixed, i.e. it can no longer be set by the user, after installation a shortcut to the logfile directory is put under the iba DatCoordinator folder in the windows menu.

Bugfixes

  • For extracting to files, report generation and the copy task, when selecting to have subfolders in de destination directory that are the same as in the source .dat directory, the DatCoordinator could accidentely delete subfolders from the parent directory of the destination directory instead of cleaning up subfolders of the destination directory. This could only happen if there where .dat files in the source .dat directory that weren't in subfolders. If the destination directory was a shared network map, it caused a crash instead of actual deletion of folders.
