TKS-Xml Plugin for iba DatCoordinator

 

Installation

Have ibaDatCoordinator installed. Required version: v2.0.0

 

Version history

 

Version 2.1.0 - 2018-02-05

Improvements (I)

  1. Changed group name resolver to a more generic version.
  2. Add second underscore to prefix "MI__" for "Messinstrument" names.

Bug fixes (B)

  1. "Einzelwerte" were not resolved properly when comment1 or comment2 were defined for names.

 

Version 2.0.0 - 2017-09-14

Improvements (I)

  1. Use ibaFilesLiteDotNet.dll again.
  2. Adjust installer for datco 2.0

 

Version history

 

Version 1.3.2 - 2015-05-20

Bug fixes (B)

  1. Infofields from first exported file were written to all following files

 

Version 1.3.1 - 2015-04-17

Bug fixes (B)

  1. New ibaFiles .Net wrapper caused hanging of ibaDatCoordinator until service was restarted

 

Version 1.3.0 - 2015-03-19

New features (N)

  1. Export single values if signal name contains "__IE__" or "__SE__"
  2. Location "SI" could be definied on option page
  3. Field TKSIDENT will be exported if defined, not empty and not 0, field BUNDNR otherwise
  4. Xml type "long" will not be checked for field TKSIDENT

 

Version 1.2.0 - 2014-09-12

New features (N)

  1. Switched to version 4.0 of the .NET framework

 

Version 1.1.0 - 2014-07-08

New features (N)

  1. Changed date export format to: 2014-06-22T00:29:42+02:00
  2. When xsd schema file "Messung.xsd" is found at given location (see config UI) then its values for units will be used also
  3. Field "Messung.LetzteMessungAmDurchsatz" will be set to true for last "Messung", false for all others
  4. If signal name ends with "__IR__S" or "__GR__S", then "Messung.Gruppe" will be set to sinal name stripped by this suffix, signal name for all others

Bug fixes

  1. Units were not detected correctly, if they contain special chars like "/"

 

Version 1.1.0 - 2013-10-18

New features (N)

  1. Additional values can be defined for "Standort"
  2. Additional values could be defined for "Schemalocation"
  3. Export floating point values with single precision

 

1.0.0 - 2013-05-28

New features (N)

  1. Initial version
